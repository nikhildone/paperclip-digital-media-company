# QA and Approval Rules

## Core Rule
No output is upload-ready until QA passes and human approval is recorded.

## Brand QA
Every output must pass:

- Clear SINK/DINK India fit.
- Clear no-kids / childfree / couple timeline / family pressure / life planning signal.
- No parenting-topic drift.
- Respectful toward families, parents, and kids.
- No insulting tone.

## Media QA
Every media pack must include:

- final media file or fallback reason
- caption
- hashtags
- cover
- voiceover text/audio
- QA report
- approval note

## Rejection Reasons
QA must reject output if:

- It becomes parenting advice.
- It becomes kids activity content.
- It loses SINK/DINK positioning.
- It is disrespectful.
- It is generic AI content with no audience insight.
- It is not upload-ready.

## Approval States
Allowed states:

- draft
- qa_pending
- qa_failed
- approval_pending
- approved_for_manual_upload
- rejected
- archived

## Human Approval
The user must approve before upload. No auto-upload is allowed in the testing project.

## QA Output Format
QA report must include:

- Verdict: pass/fail
- Brand fit
- Drift check
- Safety note
- Upload readiness
- Fix needed
- Learning note
